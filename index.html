Using % and .format() and format! for great good!

Python has had awesome string formatters for many years but the documentation on them is far too theoretic and technical. With this site we try to show you the most common use-cases covered by the old and new style string formatting API with practical examples.

If not otherwise stated all examples work with Python 2.7, 3.2, 3.3, 3.4, and 3.5 without requiring any additional libraries or monkey-patching.

Further details about these two formatting methods can be found in the official Python documentation:

  • old style
  • new style

Rust has awesome string formatter, too. The documentation explains it very well. With this site we try to compare the most common use-cases with Python's formatters.

If you want to contribute more examples, feel free to create a pull-request on Github!

Basic formatting

Simple positional formatting is probably the most common use-case. Use it if the order of your arguments is not likely to change and you only have very few elements you want to concatenate.

Since the elements are not represented by something as descriptive as a name this simple style should only be used to format a relatively small number of elements.

Python Old

'%s %s' % ('one', 'two')

Python New

'{} {}'.format('one', 'two')

Rust

format!("{} {}", "one", "two")

Output

one two

Python Old

'%d %d' % (1, 2)

Python New

'{} {}'.format(1, 2)

Rust

format!("{} {}", 1, 2)

Output

1 2

With Python's new style formatting it is possible (and in Python 2.6 even mandatory) to give placeholders an explicit positional index.

This allows for re-arranging the order of display without changing the arguments.

This operation is not available with Python old-style formatting.

Python New

'{1} {0}'.format('one', 'two')

Rust

format!("{1} {0}", "one", "two")

Output

two one

Value conversion

The Python's new-style simple formatter calls by default the __format__() method of an object for its representation. If you just want to render the output of str(...) or repr(...) you can use the !s or !r conversion flags.

In %-style you usually use %s for the string representation but there is %r for a repr(...) conversion.

This operation is not available with Rust formatting.

Setup

class Data(object):

    def __str__(self):
        return 'str'

    def __repr__(self):
        return 'repr'

Python Old

'%s %r' % (Data(), Data())

Python New

'{0!s} {0!r}'.format(Data())

Output

str repr

In Python 3 there exists an additional conversion flag that uses the output of repr(...) but uses ascii(...) instead.

This operation is not available with Python old-style formatting.

This operation is not available with Rust formatting.

Setup

class Data(object):

    def __repr__(self):
        return 'räpr'

Python New

'{0!r} {0!a}'.format(Data())

Output

räpr r\xe4pr

In Rust there exists an additional conversion flag that can use for who implmeneted std::fmt::Debug trait:

This operation is not available with Python old-style formatting.

This operation is not available with Python new-style formatting.

Rust

format!("{:?}", vec![1, 2, 3])

Output

[1, 2, 3]

Padding and aligning strings

By default values are formatted to take up only as many characters as needed to represent the content. It is however also possible to define that a value should be padded to a specific length.

Unfortunately the default alignment differs between Python's old and new style formatting. The old style defaults to right aligned while for new style it's left.

Align right:

Python Old

'%10s' % ('test',)

Python New

'{:>10}'.format('test')

Rust

format!("{:>10}", "test")

Output

      test

Align left:

Python Old

'%-10s' % ('test',)

Python New

'{:10}'.format('test')

Rust

format!("{:10}", "test")

Output

test

By argument:

In the previous example, the value '10' is encoded as part of the format string. However, it is possible to also supply such values as an argument.

Python Old

'%*s' % ((- 8), 'test')

Python New

'{:<{}s}'.format('test', 8)

Rust

format!("{:<1$}", "test", 8)

Output

test

Again, Python's new style formatting surpasses the old variant by providing more control over how values are padded and aligned.

You are able to choose the padding character:

This operation is not available with Python old-style formatting.

Python New

'{:_<10}'.format('test')

Rust

format!("{:_<10}", "test")

Output

test______

And also center align values:

This operation is not available with Python old-style formatting.

Python New

'{:^10}'.format('test')

Rust

format!("{:^10}", "test")

Output

   test

Truncating long strings

Inverse to padding it is also possible to truncate overly long values to a specific number of characters.

The number behind a . in the format specifies the precision of the output. For strings that means that the output is truncated to the specified length. In our example this would be 5 characters.

Python Old

'%.5s' % ('xylophone',)

Python New

'{:.5}'.format('xylophone')

Rust

format!("{:.5}", "xylophone")

Output

xylop

By argument:

Python Old

'%.*s' % (7, 'xylophone')

Python New

'{:.{}}'.format('xylophone', 7)

Rust

format!("{:.1$}", "xylophone", 7)

Output

xylopho

Combining truncating and padding

It is also possible to combine truncating and padding:

Python Old

'%-10.5s' % ('xylophone',)

Python New

'{:10.5}'.format('xylophone')

Rust

format!("{:10.5}", "xylophone")

Output

xylop

Numbers

Of course it is also possible to format numbers.

Integers:

Python Old

'%d' % (42,)

Python New

'{:d}'.format(42)

Rust

format!("{}", 42)

Output

42

Floats:

Python Old

'%f' % (3.141592653589793,)

Python New

'{:f}'.format(3.141592653589793)

Rust

format!("{:.6}", 3.141592653589793)

Output

3.141593

Padding numbers

Similar to strings numbers can also be constrained to a specific width.

Python Old

'%4d' % (42,)

Python New

'{:4d}'.format(42)

Rust

format!("{:4}", 42)

Output

  42

Again similar to truncating strings the precision for floating point numbers limits the number of positions after the decimal point.

For floating points the padding value represents the length of the complete output. In the example below we want our output to have at least 6 characters with 2 after the decimal point.

Python Old

'%06.2f' % (3.141592653589793,)

Python New

'{:06.2f}'.format(3.141592653589793)

Rust

format!("{:06.2}", 3.141592653589793)

Output

003.14

For integer values providing a precision doesn't make much sense and is actually forbidden in the Python's new style (it will result in a ValueError).

Python Old

'%04d' % (42,)

Python New

'{:04d}'.format(42)

Rust

format!("{:04}", 42)

Output

0042

Signed numbers

By default only negative numbers are prefixed with a sign. This can be changed of course.

Python Old

'%+d' % (42,)

Python New

'{:+d}'.format(42)

Rust

format!("{:+}", 42)

Output

+42

Use a space character to indicate that negative numbers should be prefixed with a minus symbol and a leading space should be used for positive ones.

This operation is not available with Rust formatting.

Python Old

'% d' % ((- 23),)

Python New

'{: d}'.format((- 23))

Output

-23

Python Old

'% d' % (42,)

Python New

'{: d}'.format(42)

Output

 42

Python's new style formatting is also able to control the position of the sign symbol relative to the padding.

This operation is not available with Python old-style formatting.

This operation is not available with Rust formatting.

Python New

'{:=5d}'.format((- 23))

Output

-  23

Named placeholders

Python's both formatting styles support named placeholders.

Setup

data = {'first': 'Hodor', 'last': 'Hodor!'}

Python Old

'%(first)s %(last)s' % data

Python New

'{first} {last}'.format(**data)

Rust

format!("{first} {last}", first="Hodor", last="Hodor!")

Output

Hodor Hodor!

.format() also accepts keyword arguments.

This operation is not available with Python old-style formatting.

Python New

'{first} {last}'.format(first='Hodor', last='Hodor!')

Rust

format!("{first} {last}", first="Hodor", last="Hodor!")

Output

Hodor Hodor!

Getitem and Getattr

Python's new style formatting allows even greater flexibility in accessing nested data structures.

It supports accessing containers that support __getitem__ like for example dictionaries and lists:

This operation is not available with Python old-style formatting.

This operation is not available with Rust formatting.

Setup

person = {'first': 'Jean-Luc', 'last': 'Picard'}

Python New

'{p[first]} {p[last]}'.format(p=person)

Output

Jean-Luc Picard

Setup

data = [4, 8, 15, 16, 23, 42]

Python New

'{d[4]} {d[5]}'.format(d=data)

Output

23 42

As well as accessing attributes on objects via getattr():

This operation is not available with Python old-style formatting.

This operation is not available with Rust formatting.

Setup

class Plant(object):
    type = 'tree'

Python New

'{p.type}'.format(p=Plant())

Output

tree

Both type of access can be freely mixed and arbitrarily nested:

This operation is not available with Python old-style formatting.

This operation is not available with Rust formatting.

Setup

class Plant(object):
    type = 'tree'
    kinds = [{'name': 'oak'}, {'name': 'maple'}]

Python New

'{p.type}: {p.kinds[0][name]}'.format(p=Plant())

Output

tree: oak

Datetime

Additionally Python's new style formatting allows objects to control their own rendering. This for example allows datetime objects to be formatted inline:

This operation is not available with Python old-style formatting.

This operation is not available with Rust formatting.

Setup

from datetime import datetime

Python New

'{:%Y-%m-%d %H:%M}'.format(datetime(2001, 2, 3, 4, 5))

Output

2001-02-03 04:05

Custom objects

The above example works through the use of the __format__() magic method. You can define custom format handling in your own objects by overriding this method. This gives you complete control over the format syntax used.

This operation is not available with Python old-style formatting.

This operation is not available with Rust formatting.

Setup

class HAL9000(object):

    def __format__(self, format):
        if (format == 'open-the-pod-bay-doors'):
            return "I'm afraid I can't do that."
        return 'HAL 9000'

Python New

'{:open-the-pod-bay-doors}'.format(HAL9000())

Output

I'm afraid I can't do that.
